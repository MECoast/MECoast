Note:- Only mono sound files are provided. These files can be used from within the Cab (ie stereo) sound environment provided the Stream is frequency shifted to suit. This is done to simplify the creation of the files, reduce the upload/download file sizes, and because the value of artificially 'stereoised' mono originals is not consider (in my opinion) to be worth it. Also, they play back at the correct speed but lower pitch, which makes then sound more muffled, which is more appropriate for inside the cab.

Loco Whistles:-

These whistles have been in particular developed for use with my Generic 2 Cylinder and 3 Cylinder Sound Sets. They are installed into the uk_shared folder. All of these whistles have been edited to be short (ish) and most have markers so that they will loop cleanly (well clean-ish!) whilst the spacebar is depressed. The following whistle sounds are included:-

These files all have a similar name format so changing the whistle is very easy.

Usage:-

To get a loco to play one of these whistles it will need to have both it's external view and the cab view sms files modified.

Example: To change a loco that uses myloco.eng open up this .eng file in Wordpad and note down the names of the two .sms files that it uses, for example 2c-100a-eng.sms and 2c-100a-cab.sms

Open up the first listed (external view) sms. Scroll down to find the following, or very similar:-

Stream(
       Skip( **** Whistle sounds. **** )
       Priority( 6 )
       Triggers( 2 
              Discrete_Trigger ( 8 StartLoopRelease ( 1 File ( "../UK_Shared/x_whistle_fs_01.wav" -1 ) SelectionMethod ( SequentialSelection ) ) )
              Discrete_Trigger ( 9 ReleaseLoopReleaseWithJump () )
       )
      FrequencyCurve (
              SpeedControlled
                        CurvePoints ( 2
                                 0          11000
                            999          11000
                         )
             Granularity ( 100 )
      )
)

Now to change the whistle played from fs (ie Scotsman) to, say A4, on the first Discrete_Trigger line simply change the fs in the file name to A4, and save.

Now open up the second listed (cab view) sms. Scroll down to find the following, or very similar:-

Stream(
       Skip( **** Whistle sounds. **** )
       Priority( 6 )
       Triggers( 2 
              Discrete_Trigger ( 8 StartLoopRelease ( 1 File ( "../UK_Shared/x_whistle_fs_01.wav" -1 ) SelectionMethod ( SequentialSelection ) ) )
              Discrete_Trigger ( 9 ReleaseLoopReleaseWithJump () )
       )
      FrequencyCurve (
              SpeedControlled
                        CurvePoints ( 2
                                 0          5500
                            999          5500
                         )
             Granularity ( 100 )
      )
)

Notice the only difference should be that the second column of CurvePoints should be half the figure in the external sms file, ie 5500 instead of 11000.

Now, to change the whistle played from fs (ie Scotsmans) to, say A4, on the first Discrete_Trigger line simply change the fs in the file name to A4, and save.

That's it. BUT. Simply doing the above has one big implication. If more than one loco share that particular pair of .sms files than all the locos will now have the new whistle. So, to enable multiple use of the same sound configuration but with different whistles make a copy of the original pair of .sms files ie 2c-100a-eng01.sms and 2c-100a-cab01.sms and rename them 2c-100a-eng02.sms and 2c-100a-cab02.sms ie increase the designating numbers. Then change the whistles in these files, and edit the appropriate .eng file to point to the 02 versions instead.

Note: A similar method can be used with the #AI non-drivable loco .sms files, but be careful as each whistle is referred to three times!

Pitch Shifting:-

One means of adding further variety, or even effectively a new whistle sound, is to pitch shift the whistle within the sms file. This is simply a case of editing the second figures under the CurvePoints section. Example. To pitch shift a whistle up by 10% change the 11000 figures (both) in the external view file to 11000*1.1 = 12100, and the in the cab view file to half this figure ie 6050. This would give you

      FrequencyCurve (
              SpeedControlled
                        CurvePoints ( 2
                                 0          12100
                            999          12100
                         )
             Granularity ( 100 )
      )

and

      FrequencyCurve (
              SpeedControlled
                        CurvePoints ( 2
                                 0          6050
                            999          6050
                         )
             Granularity ( 100 )
      )
